# Troubleshooting Methodology

## Overview

A structured troubleshooting approach helps solve problems faster and avoids unnecessary changes.

---

# Step 1 — Identify the Problem

Gather information.

Examples:

- What is not working?
- When did the issue start?
- Is the issue affecting one user or many?

---

# Step 2 — Establish a Theory

Possible causes:

- Loose cable
- Incorrect IP
- DNS failure
- Router failure
- Firewall blocking traffic

---

# Step 3 — Test the Theory

Example:

```
Ping Gateway
```

If it fails,

↓

Gateway problem.

If it succeeds,

↓

Check DNS.

---

# Step 4 — Create a Plan

Choose the safest solution.

Example:

- Restart Router
- Renew IP Address
- Flush DNS Cache

---

# Step 5 — Implement the Solution

Apply the fix.

Examples:

- Replace cable
- Correct IP configuration
- Restart DNS service

---

# Step 6 — Verify Functionality

Confirm:

- Internet works
- Applications work
- DNS resolves correctly
- Users can access resources

---

# Step 7 — Document the Solution

Record:

- Problem
- Cause
- Solution
- Prevention steps

Documentation helps resolve similar issues faster in the future.

---

# Troubleshooting Flow

```
Problem

↓

Identify

↓

Theory

↓

Test

↓

Fix

↓

Verify

↓

Document
```
