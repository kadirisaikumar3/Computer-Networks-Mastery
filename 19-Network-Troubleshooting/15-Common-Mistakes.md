# Common Mistakes

---

# Mistake 1

## Restarting Everything Immediately

❌ Wrong

Restart devices without checking the actual problem.

✅ Correct

Diagnose first, then apply the appropriate fix.

---

# Mistake 2

## Ignoring Physical Connections

Always check:

- Ethernet Cable
- Router Power
- Switch LEDs
- Wi-Fi Status

Many issues are caused by simple hardware problems.

---

# Mistake 3

## Assuming DNS is Always the Problem

Verify by testing:

```cmd
ping 8.8.8.8
```

If this fails, the issue is not DNS.

---

# Mistake 4

## Forgetting the Default Gateway

Without a valid gateway, remote networks cannot be reached.

---

# Mistake 5

## Ignoring Firewall Rules

Firewalls may block applications or required ports.

Always verify firewall configuration.

---

# Mistake 6

## Not Verifying the Solution

Always confirm that:

- Internet works
- Applications work
- DNS resolves correctly
- Users can access required resources

---

# Mistake 7

## Not Documenting the Issue

Document:

- Root Cause
- Solution
- Commands Used
- Prevention Steps

---

# Interview Tip

Remember

```
Identify

↓

Test

↓

Verify

↓

```
