# Frequently Asked Questions (FAQs)

## 1. What does HTTP stand for?

HyperText Transfer Protocol.

---

## 2. What does HTTPS stand for?

HyperText Transfer Protocol Secure.

---

## 3. Which layer does HTTP work on?

Application Layer.

---

## 4. Which ports are used?

HTTP → 80

HTTPS → 443

---

## 5. Why is HTTP stateless?

Because the server does not remember previous requests.

---

## 6. What is HTTPS?

HTTPS is the secure version of HTTP that uses TLS.

---

## 7. What is TLS?

Transport Layer Security.

It provides:

- Encryption
- Authentication
- Data Integrity

---

## 8. What is the difference between HTTP and HTTPS?

HTTP is unencrypted.

HTTPS encrypts communication.

---

## 9. What are HTTP Headers?

Key-value pairs that carry additional request and response information.

---

## 10. What is a Cookie?

Small data stored in the browser.

---

## 11. What is a Session?

User information stored on the server.

---

## 12. Why are Cookies required?

To identify users across multiple HTTP requests.

---

## 13. Which HTTP Method retrieves data?

GET.

---

## 14. Which HTTP Method creates a resource?

POST.

---

## 15. Which data format is commonly used in REST APIs?

JSON.

---
