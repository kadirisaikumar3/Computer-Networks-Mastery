# Introduction to HTTP

## What is HTTP?

**HTTP (HyperText Transfer Protocol)** is an application layer protocol used for communication between a **Client** and a **Server** over the Internet.

Whenever you:

- Open a website
- Search on Google
- Watch YouTube
- Shop on Amazon
- Call a REST API

HTTP is used to exchange information.

---

# Full Form

**HTTP = HyperText Transfer Protocol**

- **HyperText** → Text containing links to other resources.
- **Transfer** → Exchange of information.
- **Protocol** → A set of communication rules.

---

# Why Do We Need HTTP?

Imagine opening:

```
https://www.google.com
```

Your browser needs a standard way to communicate with Google's server.

HTTP defines:

- How requests are sent.
- How responses are received.
- How data is transferred.

Without HTTP, browsers and servers could not communicate in a standardized way.

---

# Client-Server Architecture

HTTP follows the **Client-Server Model**.

```
Client (Browser)

↓

HTTP Request

↓

Server

↓

HTTP Response

↓

Client
```

The client requests information.

The server processes the request and returns a response.

# Where is HTTP Used?

HTTP is used in:

- Websites
- REST APIs
- Mobile Applications
- Cloud Services
- Web Applications
- Microservices

Almost every modern application uses HTTP.

---

# Example

When you type:

```
www.google.com
```

the browser sends an HTTP request to Google's server.

The server responds with:

- HTML
- CSS
- JavaScript
- Images

The browser renders the webpage.
